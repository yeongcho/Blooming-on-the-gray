# 잿빛 위에 피어
- 회색 마음 위에 피어난 온기, 고양이와 함께 보내는 5일간의 이야기
- '잿빛 위에 피어'는 빗속에서 버려진 상자 속 고양이를 집으로 데려오면서 시작되는, 한 사람과 한 고양이의 5일간 동거 기록을 담은 힐링 스토리형 게임입니다.

---

## 기획 의도

- 고양이에 대한 편견과 오해를 줄이고, 고양이도 따뜻한 시선과 보호가 필요한 존재임을 전달
- 상처 입은 고양이가 회복해 가는 과정을 보여주는 힐링 스토리
- 짧은 플레이 타임 안에서 선택의 무게, 관계 맺기, 정서적 교감을 경험하게 하는 것

---

## 게임 소개

### 메인 스토리

- 비 오는 어느 날, 주인공은 버려진 골목 상자 안에서 한 마리의 고양이를 발견하고 집으로 데려온다.
- 플레이어는 5일 동안 고양이와 함께 생활하며, 매일의 선택을 통해 고양이의 마음을 얻을지, 잃을지 결정하게 된다.

### 데일리 스토리 (선택지 형식)

- Day 0 ~ Day 5로 구성된 5일의 이야기
- 하루마다 여러 개의 선택지가 등장
- 선택에 따라 다음이 변화
  - 호감도
  - 포만감

---

## 게임 플레이

### 핵심 요소

- 호감도(Affection)
  - 플레이어의 행동에 따라 증가/감소  
  - 일정 수치 이상이면 해피엔딩, 중간이면 노멀엔딩, 낮으면 배드엔딩으로 분기
    
- 포만감(Satiety)
  - 사료 주기 선택에 따라 변화
  - 선택에 따라 배드엔딩으로 이동

- 코인(Coin)
  - 미니게임 보상으로 획득
  - 사료 주기에 소비

- 색 변화(연출)
  - 고양이와의 관계가 좋아질수록 게임 화면의 색감이 점점 따뜻하게 변화  
  - 처음엔 회색톤 → 호감도에 따라 점점 컬러가 살아남

---

## 엔딩 구조

- Happy Ending
  - 호감도 상위 구간  
  - 고양이의 시점에서 주인공에 대한 신뢰와 애정을 느끼는 결말

- Normal Ending
  - 호감도 중간 구간  
  - 서로 어색하지만, 앞으로의 관계를 기대하게 되는 결말

- Bad Ending
  - 호감도 낮고 / 특정 조건(포만감 : 0) 충족 시  
  - 고양이가 집을 떠나고 관계가 멀어지는 결말

---

## 미니게임

- 20초 동안 진행되는 간단한 액션 미니게임
- 조작: 마우스로 장애물을 피하며 생존
- 성공 시
  - 호감도 보너스
  - 코인 보상
- 난이도: 게임상의 날짜가 지날수록 장애물/속도 증가

---

## 씬 구성

Unity 씬은 다음과 같이 구성되어 있습니다.

- TitleScene : 타이틀 화면 (게임 시작 / 종료 버튼)
- Day0Scene ~ Day5Scene : 스토리 데이별 이벤트
- MainScene : 메인 게임 / UI 관리
- MiniGameScene : 미니게임 전용 씬
- HappyEndingScene : 해피 엔딩
- NormalEndingScene : 노멀 엔딩
- BadEndingScene : 배드 엔딩

---

## 기능 요구사항

### 1. 공통 / 시스템

1. 타이틀 화면에서 다음 버튼을 제공해야 한다.
   - START : Day0Scene으로 전환
   - EXIT : 애플리케이션 종료
2. 모든 씬 전환은 SceneManager.LoadScene()을 통해 이루어져야 한다.
3. 텍스트 표현에는 TextMeshPro를 사용한다.

### 2. 스토리 & 선택지

1. Day0 ~ Day5까지 최소 6개의 스토리 씬을 제공해야 한다.
2. 각 Day 씬에는 3개의 선택지가 존재해야 한다.
3. 선택지 선택 시, 현재 상태값(호감도/포만감 등)이 즉시 갱신되어야 한다.
4. Day5 종료 시, 최종 상태값에 따라 엔딩 씬으로 전환된다.

### 3. 상태 관리 (호감도 / 포만감 / 코인)

1. 호감도는 0 이상의 정수로 관리한다.
2. 포만감은 최소/최대 범위를 가지며, 범위를 넘어가면 클램핑 처리한다.
3. 코인은 미니게임 성공 시 일정량 증가한다.
4. 상태값은 씬이 전환되어도 유지되어야 한다.

### 4. 연출 / 색 변화

1. 메인 게임 화면의 배경은 호감도에 따라 채도가 달라진다.

### 5. 미니게임

1. 미니게임은 별도의 `MiniGameScene`에서 진행된다.
2. 게임 시작 시, 제한 시간(20초) 카운트다운 UI를 표시해야 한다.
3. 미니 게임 플레이어 캐릭터(고양이)는 기본 이동 조작(상, 하)을 지원해야 한다.
4. 장애물 오브젝트는 일정 주기로 생성되며, 플레이어가 충돌 시 패널티로 처리한다.
5. 제한 시간 동안 생존 시 성공 처리 및 다음을 수행한다.
   - 호감도 증가
   - 코인 보상 지급
6. 실패 시, 미니 게임은 종료
   - 호감도 감소

### 6. 엔딩

1. Day5 종료 후, 다음 조건으로 엔딩을 결정한다.
   - 호감도 80이상 → HappyEndingScene
   - 호감도 50이상 ~ 80미만 → NormalEndingScene
   - 호감도 50미만 이거나 포만감 0 → BadEndingScene
2. 각 엔딩 씬은 고유한 일러스트/대사를 가진다.
3. 엔딩 씬 후에는 시작화면으로 돌아온다.
